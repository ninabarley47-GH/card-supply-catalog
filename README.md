# card-supply-catalog

*A visual library for paper crafters.*

Card Supply Catalog is an offline-first web application designed to help paper crafters organize, browse, and rediscover their Designer Series Paper (DSP) collection.

Unlike traditional inventory software, Card Supply Catalog is built around a simple idea:

> **Explore your paper collection visually.**

The application emphasizes visual browsing over data entry, making it easy to locate the perfect paper for a project without remembering product names or searching through storage cabinets.

---

## Why This Project Exists

Many paper crafters own dozens—or even hundreds—of paper packs. While keeping an inventory can be helpful, most existing solutions focus on tracking quantities, prices, or storage locations.

Card Supply Catalog takes a different approach.

Instead of asking:

* How many packs do I own?
* What did I pay?
* Where is it stored?

It answers the questions crafters actually ask while creating:

* What papers do I have available to use?
* What do they look like?
* Who does this paper belong to?

---

## Core Features

### Visual Library

Browse your collection using four pattern thumbnails for each DSP pack, making recognition fast and intuitive.

### Fast Search

Instantly search by:

* Partial product name
* Keywords
* Owner
* Release year

### Simple Cataloging

Add new DSP packs in under a minute.

The application automatically:

* Creates pattern records
* Organizes image folders
* Saves catalog information
* Tracks when items were added

### Flexible Image Management

Images can be:

* Imported from a folder
* Added individually
* Replaced at any time

DSP packs can be cataloged immediately, even if images are not yet available.

### Offline First

All data is stored locally.

Your collection remains yours.

No subscriptions.

No cloud dependency.

---

## Design Philosophy

Card Supply Catalog follows several guiding principles:

* The paper is the interface.
* Recognition is better than recall.
* Visual browsing comes before data entry.
* Every field must justify its existence.
* The application remembers whenever possible.
* The interface stays quiet so the artwork can shine.

## Technology

Card Supply Catalog is built using standard web technologies:

* HTML5
* CSS3
* Modern JavaScript (ES Modules)

The project intentionally avoids unnecessary frameworks and dependencies in favor of simplicity, maintainability, and long-term reliability.

---

## Running the application locally

The application uses JavaScript ES modules and should be served from a local HTTP server during development.

Example using Python:

    python -m http.server 8000

Then open:

    http://localhost:8000
